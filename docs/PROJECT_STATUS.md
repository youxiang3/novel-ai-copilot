# NovelAI Copilot 项目状态

更新时间：2026-06-01

## 当前判断

项目目前处在“可交互前端原型 + 后端骨架”的阶段。前端已经能展示核心创作体验，并且支持本地账号、本地作品、多作品切换、作品资料总览和创作向导；但真实后端数据闭环、真实模型能力、自动化测试和正式部署还没有完成。

后端已经有 Spring Boot、JWT、章节、设定、工作流、模型配置等骨架，但当前本机环境缺 Maven，默认 `javac` 仍是 Java 8，所以还没有在本机完成后端编译和端到端验证。

## 已完成

### 前端体验

- 首页已从空白编辑器改成“脑洞入口”，展示产品特色：脑洞拆解、黄金第一章、短画面扩写、卡文急救、长篇记忆、伏笔/OOC 检查。
- 已实现三栏创作工作台：左侧作品资产、中心正文编辑、右侧 AI 副驾。
- 左侧已加入“总览”页，集中展示作品资料、人物角色、章节概览。
- 总览页支持编辑：一句话卖点、全局大纲、主角、世界规则、长期剧情弧、当前追读线。
- 总览页会汇总人物角色卡和章节概览，解决切换作品后看不到大纲、人物、目录的问题。
- 已实现清爽浅色 UI：浅青绿主色、纸张感、轻网格、简洁工具区。
- 已实现本地账号模式：后端未启动时也能注册、登录、创建本地作品。
- 已实现本地作品持久化：本地账号下保存作品列表，每个作品保存独立章节、设定、灵感、作品资料和工作区状态。
- 已实现作品切换：切换本地作品时会保存当前作品，并加载目标作品。
- 已修复新建作品仍显示“仙途漫漫”的问题，新作品会创建空白工作区。
- 已升级新建作品流程：支持填写“新作品标题 + 一句脑洞”，创建时自动初始化作品资料、第一章开篇画面、主角设定卡、世界规则设定卡。
- 已实现章节编辑、新建章节、删除章节、字数统计、草稿/发布状态展示。
- 已实现设定库基础 CRUD、搜索、分类、标签、角色立绘 Prompt 字段。
- 已实现灵感记录、删除、回填到短画面输入。
- 已实现选中文本后的润色、扩写、重写指令入口。
- 已实现 Scene-to-Chapter 短画面扩写入口，后端不可用时有本地 fallback。
- 已实现右侧 AI 副驾：责编、军师、搭子三种模式的原型交互。
- 已实现创作向导：可用于新书开局，也可用于当前章节卡文。
- 已实现创作向导动态选项：选项会根据前面回答和当前章节内容变化。
- 已实现卡文场景下生成“续写方案 + 可直接续写段落”，并追加到当前章节。
- 已实现 IP 衍生工具原型：短剧脚本、章节摘要、责编审稿、角色立绘 Prompt。
- 已实现 Prompt Inspector 原型：文风、近期记忆、实体召回、当前指令。
- 已实现 Codex 协作模式：没有模型 API 时可复制任务包，让 Codex 代替模型生成，再粘回页面。

### 模型与 API

- 前端已提供模型 API 设置入口：Provider、Base URL、模型名、API Key。
- 已支持 DeepSeek、OpenAI、硅基流动、OpenRouter、自定义等 OpenAI-compatible 配置。
- 已实现模型测试入口。后端未登录时也可临时测试 API Key；保存配置仍要求账号。
- 后端已添加用户模型配置表 `user_model_config`。
- 后端已添加 API Key 加密/解密服务。
- 后端 `AIServiceImpl` 已支持优先使用用户模型配置调用 OpenAI-compatible `/chat/completions`。
- Next.js 已添加 `/api/*` 到 `localhost:8080/api/*` 的 rewrite。

### 后端骨架

- 已有 JWT 拦截器。

- 已补章节和设定的用户归属校验。

## 未完成

### 产品体验

- 新建作品流程已有“标题 + 脑洞 -> 初始化作品资料”的前端原型，但还没有“用户确认草稿后再创建”的分步确认流程。
- 章节目录还没有拖拽排序、分卷、章节状态流转和批量管理。
- 设定库还没有完整建模为人物、地点、物品、势力、规则、事件、功法等细分类。
- 人物角色还没有单独的人物卡视图、关系图、成长弧、出场章节记录。
- 灵感系统还没有分类、关联设定、转剧情树。
- Prompt Inspector 还没有 token 预算、模型参数、真实 Prompt 拼装详情。

### AI 能力

- 前端很多 AI 功能仍是本地 fallback 或演示逻辑，没有全部接真实模型。
- Scene-to-Chapter 还没有完整接入后端记忆、Lore、文风、章节目标。
- 创作向导目前是规则生成，不是真正的模型驱动问答。
- AI 副驾聊天是原型回复，还没有真实多轮上下文。
- 多模型对比生成尚未实现。
- 文风克隆尚未实现。
- RAG/向量召回尚未实现。
- OOC 检测、伏笔检测、节奏检测尚未真实接模型分析。
- 短剧/漫画/角色立绘目前主要是 Prompt 或文本原型，尚未接图像模型或短视频生产链路。

### 后端与数据

- 后端尚未在当前环境完成编译验证。当前机器缺 Maven，默认 `javac` 是 Java 8，而项目需要 Java 17+。
- PostgreSQL 数据库和 schema 尚未在当前环境验证。
- Redis 依赖尚未验证。
- 登录、注册、章节保存、设定保存、模型配置保存需要后端运行后做端到端测试。
- 本地账号数据目前保存在浏览器 localStorage，不是正式账号系统。
- 后端流式模型调用目前不是严格逐 token 流式，只是兼容 SSE 响应解析的雏形。
- 缺少数据库迁移工具，当前依赖 `schema.sql`。

### 工程化

- 缺少自动化测试。
- 缺少 CI。
- 缺少正式部署配置。
- 缺少环境变量校验和启动前检查。
- 缺少错误边界、空状态和加载状态的系统化处理。
- `PRODUCT_DOCUMENT.md` 仍需要按当前原型重新整理成最新版 PRD。

## 下一步建议

### 第 1 步：新建作品确认流程

目标：把当前“一键创建并初始化”升级成“先生成作品资料草稿，用户确认后再创建”。

建议流程：

1. 用户输入作品名和一句脑洞。
2. 创作向导生成作品资料草稿。
3. 用户可编辑卖点、大纲、主角、世界规则。
4. 用户确认后创建作品。
5. 自动生成第一章空草稿、主角 Lore、世界规则 Lore、前三章推进。

### 第 2 步：接真实模型链路

目标：让产品从“原型”变成“真的能写”。

优先打通：

- 模型 API 测试
- 短画面扩写
- 创作向导生成作品资料
- 卡文急救

### 第 3 步：补后端环境

目标：让注册、登录、保存作品、保存章节、保存设定进入真实数据库。

需要：

- Java 21 或 Java 17
- Maven
- PostgreSQL
- Redis
- 执行 `schema.sql`
- 启动 Spring Boot 后端

### 第 4 步：整理最新版 PRD

目标：让 `PRODUCT_DOCUMENT.md` 从早期设想升级成和当前产品一致的最新版产品文档。

建议补齐：

- 当前首页定位
- 本地账号与正式账号差异
- 作品资料总览
- 新建作品向导
- 模型 API 接入策略
- Codex 协作兜底方案
- MVP 与后续版本边界

## 当前最重要的未完成点

当前最影响体验的是“真实模型链路”和“新建作品确认流程”。

作品资料总览已经补上，用户切换作品后可以看到目录、大纲、人物角色和剧情线。新建作品也已经能用标题和脑洞初始化作品资料。下一步应优先让创作向导接真实模型，并把新建作品做成可确认、可编辑的分步流程。
