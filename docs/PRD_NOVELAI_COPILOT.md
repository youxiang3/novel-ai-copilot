# NovelAI Copilot PRD

## 产品定位

NovelAI Copilot 是面向中文长篇小说 / 网文创作者的本地优先创作工作台。产品必须保留基础 AI 写小说能力，包括脑洞整理、大纲生成、章节续写、扩写、改写、摘要、检查和卡文急救；但核心定位不是替代作者自动写完整本书，而是把作品、章节、设定、故事线、提示词、长篇记忆、追读诊断和辅助修稿集中到一个统一的项目管理环境中。

产品差异化方向：从“AI 帮我生成正文”升级为“像网文编辑一样帮助作者把章节改到更能追读”。基础生成是入口，追读诊断、人味修订、长篇记忆和一致性检查是长期壁垒。

当前实现是前端 localStorage 闭环，不是后端数据库持久化；云同步、真实模型调用和真实平台发布均未完整接入。

## 目标用户

- 网络小说作者：需要管理多部作品、章节进度、发布平台和连载状态。
- 长篇故事创作者：需要维护人物、世界观、伏笔、全局大纲和章节推进。
- 使用外部 AI 的作者：需要复制 Prompt 到网页 AI，并把生成结果粘贴回产品解析。
- 已经能写但需要改稿的作者：需要判断章节是否有追读、是否太水、是否 AI 腔、是否缺少章末钩子。
- 早期产品验证用户：需要验证创作项目管理、作品驾驶舱和本地创作闭环是否成立。

## 当前阶段目标

- 建立统一 App Shell：左侧导航、顶部栏、中间工作区、右侧详情和工具区。
- 完成作品库 / 项目管理页 P0：让作品从简单列表升级为创作项目管理仪表盘。
- 完成作品总览 / 作品驾驶舱 P0：进入一本作品后能看到作品身份、进度、资料摘要、章节入口和官方工具状态。
- 完成本地端到端验收：新建作品、作品管理、作品总览、章节基础编辑和 localStorage 持久化可以串起来。
- 保持基础写作链路可用：脑洞 -> 资料草稿 -> 章节正文 -> 摘要 / 检查 / 改写建议 -> 保存。
- 新增追读修稿方向的产品定义：追读诊断、人味修订和修改建议作为后续核心官方工具。

## 核心用户流程

1. 首页 / 脑洞入口创建作品。
2. 输入标题和脑洞，生成或整理作品资料草稿。
3. 用户确认创建后，作品写入本地作品库。
4. 在作品库中搜索、筛选、排序、切换卡片 / 列表视图。
5. 打开作品管理弹窗，编辑作品名、笔名、状态、目标字数、发布平台、简介和标签。
6. 点击作品进入作品总览，查看身份区、核心信息、全局大纲、进度、资料摘要、章节入口和官方工具状态。
7. 从总览进入正式写作工作台、章节编辑、设定库、长篇记忆、OOC 检查或故事线图。
8. 使用外部网页 AI 时，复制 Prompt 到外部工具，再把结果粘贴回产品解析。
9. 对当前章节运行追读诊断，查看开头钩子、主角目标、冲突压力、爽点兑现、信息密度和章末悬念。
10. 对当前章节运行人味修订，识别模板腔、总结腔、过度解释、人物口吻不自然和剧情过顺的问题。
11. 用户选择修改建议，复制、插入、替换选中文本或保存为修稿记录，所有改动必须由用户确认。

## 新增核心方向：追读修稿工具

新增工具不命名为“反 AI 味检测器”，避免产品显得像规避检测或洗稿工具。建议统一命名为“追读诊断”或“章节修稿台”，内部包含：

- 追读评分：开头钩子、主角目标、冲突压力、爽点兑现、信息密度、章末悬念、下一章驱动力。
- 人味检查：模板腔、总结腔、过度解释、句式过齐、人物台词不像人、剧情过于顺滑、价值升华过硬。
- 爆点建议：指出当前章节缺少的情绪波峰、反转、压迫感、爽点或期待差。
- 修改建议：给出问题位置、影响原因、修改方向和可替换示例。
- 多版本改法：保守改、强刺激改、爽文改，用户手动选择是否插入。

这些功能服务于中文网文作者的“追读”和“改稿”需求，是区别于通用 AI 生成器的核心能力。

## 优先级

### P0

- 稳定作品库、作品管理弹窗、作品总览和章节基础链路。
- 保证 localStorage 读写、刷新后恢复和缺省字段兜底。
- 明确所有原型工具和模型待接入状态，避免误导用户。
- 补齐基础产品 / 开发文档。

### P1

- 后端作品、章节、设定和故事线持久化。
- 账号登录和真实云同步。
- Prompt 模板库和可编辑外部生成 Prompt。
- 章节数据模型拆分和作品统计联动。
- 正式接入模型调用、记忆抽取和一致性检查。
- 追读诊断 V0：先基于本地规则和 Prompt 输出追读维度，不承诺真实平台数据预测。
- 人味修订 V0：减少模板腔和说明腔，增强人物口吻、动作细节和不确定性。
- 安全的导出能力。

### P2

- 真实发布平台对接。
- 多设备协作和版本历史。
- 深度长篇记忆、人物关系推理和伏笔追踪。
- 基于长篇记忆、章节表现和用户反馈的追读诊断增强。
- 章节修稿历史和多版本对比。
- 商业化、团队协作、社区和运营能力。

## 当前限制和风险

- localStorage 数据可能因清理浏览器缓存、更换设备或隐私模式而丢失。
- 当前同步状态多为 UI 占位，不代表真实云端同步完成。
- 官方工具中部分能力只是入口、原型展示或模型待接入。
- 追读诊断只能提供创作建议，不代表真实平台追读率预测。
- 人味修订用于提升文本自然度和作者风格，不应承诺规避任何检测系统。
- 作品字数、章节数、月更新数当前依赖前端字段和轻量统计，尚未由后端统一计算。
- 复制、导出、导入备份、归档、删除已在本地闭环内实现；仍不是后端持久化、云端备份或跨设备恢复。
- 旧文件中可能仍存在历史中文编码残留，需要后续逐步清理。
