# 本地 E2E 验收清单

## 启动方式

```bash
cd frontend
npm run dev
```

打开：

`http://localhost:3000`

代码变更后建议运行：

```bash
cd frontend
npm run build
```

如果 build 遇到 `.next/trace` 被占用，通常是 dev server 正在运行，需要先停止本地开发服务再构建。

## 新建作品验收

- 从首页 / 脑洞入口创建新作品。
- 输入标题和脑洞。
- 生成或整理作品资料草稿。
- 用户确认创建。
- 新作品进入作品库。
- 新作品可以进入作品总览。
- 字段缺失时不出现 NaN、undefined、null 或页面报错。
- 游客草稿显示“仅本地保存”相关提示。

## 作品管理验收

- 打开作品管理弹窗。
- 修改作品名。
- 修改笔名。
- 修改项目状态。
- 修改当前发布状态。
- 修改目标字数。
- 修改计划章节数。
- 修改本周更新目标。
- 修改发布平台。
- 修改简介。
- 修改标签。
- 保存后卡片视图同步刷新。
- 保存后列表视图同步刷新。
- 保存后右侧当前作品详情同步刷新。
- 保存后顶部统计同步刷新。
- 搜索、筛选、排序结果能反映最新数据。
- 刷新浏览器后数据仍存在。

## 作品总览验收

- 从作品库点击进入作品。
- 默认进入作品总览 / 作品驾驶舱。
- 总览展示最新作品数据。
- 作品管理弹窗保存后的字段能在总览同步显示。
- 全局大纲默认显示前 4 条。
- 超过 4 条时可以展开 / 收起。
- 没有全局大纲时显示空状态。
- 没有人物、世界规则、章节时显示空状态。
- 创作进度不出现 NaN、undefined、null。
- 游客草稿和正式作品提示清楚。
- 官方工具状态不误导用户以为真实模型已经完整接入。

## 章节管理验收

- 从总览进入章节管理。
- 新建章节。
- 编辑章节标题和正文。
- 保存章节。
- 字数统计正常。
- 删除章节仍正常。
- 发布标记仍正常。
- 返回作品库后章节数、字数、最近更新时间尽量保持合理展示。
- 刷新浏览器后章节数据不丢失。

如果发现章节统计和作品库统计没有完全联动，先记录风险，不要大改章节管理逻辑。

## localStorage 验收

确认以下 key 没有被重命名、清空或迁移：

- `yixie-works-library-v1`
- `yixie-works-library-view-v1`
- `yixie-storyline-data-v1-${workId}`
- `yixie-phase3-workspace-${workId}`

验收步骤：

- 创建或编辑作品后刷新页面。
- 确认作品库仍能读取作品。
- 确认作品总览读取同一份数据。
- 切换卡片视图 / 列表视图后刷新，确认视图模式保留。
- 进入故事线图，确认对应作品的数据读取正常。

## 工具入口验收

确认以下入口不被破坏：

- WorkManagementModal
- 新建作品确认流程
- 章节管理
- 短画面扩写入口
- 卡文急救入口
- 世界观图谱入口
- 故事线图入口
- 长篇记忆入口
- OOC 检查入口

## 已知限制

- 当前是前端 localStorage 闭环，不是后端持久化。
- 当前不是完整云同步。
- 当前没有真实模型 API 调用。
- 复制、导出、导入备份、归档、删除已在本地闭环内实现；仍不是云端备份或跨设备恢复。
- 部分工具是原型展示或模型待接入。
- 章节统计和作品库统计仍可能存在轻微不同步。
- 移动端完整适配尚未完成。
