# NovelAI Copilot 产品重构主线

更新时间：2026-07-10

## 新定位

NovelAI Copilot 是面向中文网文作者的长篇创作管理、内容诊断和 Agent 协作平台。

它不以“再提供一个 AI 聊天框”作为核心价值，而是管理整部小说持续变化的项目状态：章节、版本、人物、关系、世界规则、时间线、故事线、伏笔、剧情承诺、长篇记忆、章节任务和诊断证据。

一句话分工：

- GPT 负责思考、表达和方案探索。
- Codex 负责读取项目、执行修改、验证结果和留下可追溯记录。
- NovelAI Copilot 负责保存、约束和调度整部小说的真实状态。

产品主张：写到一百万字，AI 依然记得你的故事。

## 核心闭环

1. 作者在作品驾驶舱看到当前阶段、未解决问题和下一章目标。
2. 系统基于大纲、上一章、人物状态、伏笔和已确认记忆生成下一章任务卡草稿。
3. 作者确认核心目标、情绪目标、必须完成、禁止事项和节奏步骤。
4. 作者或外部 AI 在任务约束下完成章节。
5. 系统给出带原文证据的追读、一致性和表达诊断。
6. 作者选择修改建议，生成候选改稿并查看 diff。
7. 作者确认采用后保存章节版本，并提取待确认记忆。
8. 已确认记忆和状态变化成为下一章任务卡的输入。

## 核心模块

- 作品驾驶舱：进度、阶段、风险、待办和最近变更。
- 章节编辑器：正文、任务约束、诊断、改稿、版本与恢复。
- 下一章任务卡：把大纲和长期状态转换成一章可执行的写作要求。
- 长篇记忆：候选、来源、确认、修正、过期与召回。
- 证据化诊断：每个问题必须给出文本证据、原因、优先级和修改方向。
- 人物与关系状态：记录人物在不同章节后的状态变化。
- 伏笔与剧情承诺：记录埋设、强化、回收、超期和影响范围。
- 创作阶段状态机：规划、写作、诊断、改稿、确认、发布准备。
- Agent 协作中心：执行计划、权限确认、运行记录和结果回填。
- Codex 适配：标准项目目录、`AGENTS.md`、CLI、Skill、MCP 和执行审计。

## 数据模型演进

核心实体按以下顺序落地：

1. `Work`、`Chapter`、`ChapterVersion`。
2. `ChapterTask`、`AIAction`。
3. `Memory`、`MemoryCandidate`。
4. `DiagnosticRun`、`DiagnosticIssue`。
5. `CharacterState`、`Relationship`、`WorldRule`、`TimelineEvent`。
6. `Storyline`、`StoryBeat`、`Foreshadowing`、`StoryPromise`。
7. `AgentExecution`、`ExportSnapshot`。

## 实施优先级

### P0 创作闭环

- 多章节基础和章节版本模型正式化。
- 下一章任务卡 V0。
- 长篇记忆候选、确认、来源和过期流程 V0。
- 带证据的章节诊断卡 V0。
- 任务卡 -> 写作 -> 诊断 -> 改稿 -> 版本 -> 记忆闭环。

### P1 Codex 协作

- 标准小说项目目录导出。
- 自动生成 `AGENTS.md` 和作品上下文文件。
- NovelAI CLI、Codex Skill、MCP Server。
- 操作权限、执行记录和结果回填。

### P2 专业能力

- 人物关系状态、伏笔承诺、时间线和创作阶段状态机。
- 后端语义检索和稳定的 Agent 多步编排。
- 完整云同步、冲突检测和差异合并。

### P3 外部生态

- IP 资产管理、生图、分镜和可运行游戏工程导出。
- 发布平台、团队协作和商业化能力。

## 当前边界

- 下一章任务卡前后端 V0 已闭环：`chapter_task` 支持章节绑定、当前版本、历史版本、状态流转、来源依据和工作台同步；本地作品仍保留快照兜底。
- 章节版本前后端 V0 已闭环：`chapter_version` 支持创建 / 列表 / 详情 / 安全恢复；保存当前章节、手动留版本和确认采用 AI 改稿会分别写入可追溯版本，编辑器可查看并恢复当前章节版本。
- 长篇记忆前后端 V0 已闭环：`memory_entry` 支持候选、确认、拒绝、过期、来源章节、原文证据和批量同步；真实模型抽取、语义召回和冲突合并仍需继续建设。
- 证据化诊断前后端 V0 已闭环：`diagnostic_run` / `diagnostic_issue` 统一保存运行历史、原文证据、原因、建议、风险统计和处理状态；真实模型诊断和跨运行对比仍需继续建设。
- 统一创作阶段状态 V0 已接入：任务规划、写作、诊断、改稿、确认、记忆、发布准备和完成均有后端历史及工作台自动触发。
- 下一步进入模型化增强：记忆抽取、一致性诊断、语义召回和 Agent 多步编排。
- IP、生图和游戏导出保留现有入口，但不再排在长篇创作闭环之前。
