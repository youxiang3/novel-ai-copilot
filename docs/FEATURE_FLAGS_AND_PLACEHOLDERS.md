# 功能状态与占位清单

## 目的

本文档用于区分 NovelAI Copilot 当前哪些能力已经本地可用，哪些只是 UI 占位、原型展示、模型待接入或云端待同步，避免后续开发和产品文案误导用户。

当前产品仍是前端 localStorage 闭环，不是后端数据库持久化，没有完整云同步，也没有真实模型调用。

## 状态分类

| 状态 | 含义 | 展示原则 |
| --- | --- | --- |
| 本地可用 | 在当前浏览器中可以真实使用 | 明确说明仅本地保存 |
| UI 占位 | 页面或按钮存在，但真实逻辑未实现 | 按钮禁用或标记“待开放” |
| 原型展示 | 有静态或本地演示逻辑 | 不承诺真实模型或云端能力 |
| 模型待接入 | 需要后续接入真实模型 | 不写“已检查”“已生成”“已记住” |
| 云端待同步 | 需要后端和同步服务 | 不写“云端已同步” |
| 后端待接入 | 需要数据库和接口 | 不承诺多设备可用 |

## 同步状态文案

当前没有完整云同步。推荐展示：

| syncState | 推荐文案 |
| --- | --- |
| `local-only` | 仅本地保存 |
| `pending` | 等待同步，云端能力待接入 |
| `syncing` | 同步中，当前为原型状态 |
| `synced` | 本地已保存，不展示为云端已同步 |
| `failed` | 同步失败，真实云同步待接入 |

如果没有真实同步服务，不建议向用户展示“云端已同步”。

## 禁止误导文案

不要写：

- 已云端同步。
- 已发布到平台。
- 已自动发文。
- 长篇记忆已建立。
- AI 已记住全书。
- OOC 检查通过。
- 人物一致性良好。
- 世界观已持久化。
- 爆款概率。
- 必火。
- 去 AI 检测。
- 自动洗稿。
- 已生成可商用封面。

推荐写：

- 本地已保存。
- 仅保存在当前浏览器。
- 云端同步待接入。
- 模型待接入。
- 原型展示。
- 需要用户确认后写入。
- 爆款潜力诊断，仅供修稿参考。
- 去模板腔 / 增强作者表达。
- 生图待接入或仅作为视觉参考。

## 功能上线前检查

任何从占位进入真实可用的功能，都需要确认：

- 是否有真实数据写入。
- 是否有失败状态。
- 是否有撤销或恢复策略。
- 是否有用户确认。
- 是否会影响 localStorage 旧数据。
- 是否需要更新 `docs/PROJECT_STATUS.md`。
