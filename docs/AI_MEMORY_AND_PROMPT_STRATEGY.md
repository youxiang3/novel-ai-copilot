# 长篇记忆与 Prompt 策略

## 战略判断

用户选择长篇记忆作为官方工具的最高战略优先级。原因是长篇小说创作的核心痛点不是单次生成，而是长期上下文、人物一致性、设定延续和伏笔回收。

但当前阶段不直接实现完整 MemoryExtractionSkill。正确路径是先做长篇记忆 V0 前置能力，建立可确认、可追溯、可修正的记忆条目。

当前已有部分后端模型调用能力（例如 API 改稿、IP Factory MVP 和模型配置），但本文讨论的长篇记忆抽取、向量库检索和后端记忆持久化仍未完整接入。

新增产品方向：追读诊断、人味修订和修改建议应建立在章节文本、作品资料、长篇记忆和用户确认机制之上。它们不是“反 AI 检测”工具，而是章节修稿工具：帮助作者发现追读弱点、模板腔和人物表达问题，并提供可确认的改稿方案。

## 长篇记忆分期

### V0：本地记忆前置能力

目标：让用户能看到、确认和维护“本书的重要记忆”。

能力范围：

- 章节摘要。
- 事件摘要。
- 人物状态摘要。
- 伏笔 / 设定摘录。
- 来源章节记录。
- 用户确认后写入。
- 标记记忆来源和更新时间。

V0 不承诺：

- AI 已记住全书。
- 自动长期上下文加载。
- 自动 OOC 检查。
- 自动纠错。
- 真实模型抽取。

### V1：模型辅助抽取

目标：接入真实模型，辅助从章节中提取候选记忆。

流程：

1. 用户选择章节或当前正文。
2. 系统构造 Prompt。
3. 模型返回候选记忆。
4. 用户确认、修改或拒绝。
5. 确认后的条目写入 MemoryItem。

原则：

- AI 只生成候选。
- 用户确认后才写入。
- 记忆条目必须有来源。
- 不自动覆盖已有设定。

### V2：MemoryExtractionSkill

目标：形成正式长篇记忆抽取能力。

能力：

- 人物、事件、设定、伏笔、世界规则分类抽取。
- 来源章节和原文片段引用。
- 置信度。
- 过期检测。
- 与故事线图、人物关系图、OOC 检查联动。

V2 依赖：

- 稳定章节数据。
- 后端持久化。
- Prompt 上下文结构稳定。
- 用户确认机制。

## OOC 与长篇记忆的关系

OOC 检查不应早于长篇记忆成熟。

OOC 至少依赖：

- 人物设定。
- 人物关系。
- 历史行为。
- 当前章节内容。
- 已发生事件。
- 世界规则。
- 长篇记忆。

如果没有这些数据，OOC 检查容易变成泛泛而谈。因此当前 OOC 应显示为原型展示或模型待接入。

## 追读诊断与人味修订

追读诊断、人味修订和修改建议是中文网文向的章节级修稿能力。它们与长篇记忆和 OOC 检查并列，但目标不同：

- 长篇记忆解决“前后是否记得住”。
- OOC / 一致性检查解决“有没有写崩”。
- 追读诊断解决“读者有没有理由继续看”。
- 人味修订解决“文本是否太模板、太说明、太像机器写”。

### 追读诊断 V0

输入：

- 当前章节标题和正文。
- 作品题材、卖点、简介。
- 当前章节摘要和下一步目标。
- 已确认的人物、世界观、伏笔和记忆条目。

输出：

- 追读评分，不显示为真实平台数据。
- 开头钩子问题。
- 主角目标清晰度。
- 冲突压力和损失感。
- 情绪波峰和爽点兑现。
- 信息密度和水段。
- 章末悬念和下一章驱动力。
- 位置级修改建议。

禁止：

- 不承诺真实追读率。
- 不自动覆盖正文。
- 不把建议写成平台算法结论。
- 不使用“必爆”“必火”等误导性文案。

### 人味修订 V0

输入：

- 当前章节正文。
- 作者选择的修订强度。
- 作品风格和人物口吻。
- 用户确认过的历史改稿偏好。

诊断维度：

- 模板化句式。
- 总结腔 / 说明腔 / 价值升华腔。
- 过度解释人物心理。
- 人物台词不像人。
- 动作细节不足。
- 剧情过顺，缺少误判、欲望、犹豫和反差。

输出：

- 问题片段。
- 问题原因。
- 保守改。
- 强刺激改。
- 爽文改。
- 可复制 / 可插入 / 可替换的文本建议。

边界：

- 不命名为“绕过 AI 检测”。
- 不做批量洗稿。
- 不自动替换全章。
- 所有修改必须由用户确认。

### 修改建议数据结构

RevisionSuggestion：

| 字段 | 含义 |
| --- | --- |
| `id` | 建议 ID |
| `workId` | 所属作品 |
| `chapterId` | 所属章节 |
| `type` | readthrough / human-taste / ooc / polish |
| `severity` | high / medium / low |
| `sourceText` | 原文片段 |
| `problem` | 问题说明 |
| `reason` | 为什么影响追读或自然度 |
| `suggestion` | 修改方向 |
| `variants` | 多版本改法 |
| `status` | draft / applied / dismissed |
| `createdBy` | local-rule / model / user |
| `createdAt` | 创建时间 |

RevisionSuggestion 可以先保存在本地生成记录中，后续再进入后端版本历史和 Prompt Inspector。

## Prompt 策略

用户选择 Prompt 最终进入 Prompt Inspector。

分期如下：

### P0.5：本次 Prompt 可编辑

能力：

- 默认 Prompt 展示。
- 用户本次编辑 Prompt。
- 导入 `.txt` / `.md` Prompt。
- 恢复默认。
- 复制当前 Prompt。
- 不保存为模板。
- 不修改提示词库。
- 不接真实模型。

适用场景：

- 外部网页 AI。
- 临时实验。
- 快速调整生成要求。

### P1：Prompt 轻工作流

能力：

- 展示上下文来源。
- 展示 Prompt 摘要。
- 用户补充要求。
- 生成前确认。
- 结果回填后用户选择插入、复制或放弃。
- 记录最近若干次 Prompt。

仍不做：

- 复杂模板库。
- 模板变量编辑器。
- Prompt 市场。
- 自动覆盖正文。

### P2：Prompt Inspector

能力：

- 显示 Prompt 完整内容。
- 显示上下文来源。
- 显示引用的章节、人物、世界规则和记忆条目。
- 显示变量替换结果。
- 显示 token 估算。
- 显示风险提示，例如上下文不足、记忆过期、引用冲突。
- 支持调试 Prompt。

Prompt Inspector 是正式 AI 能力接入后的调试和透明层，不应在数据结构未稳定前强行实现。

## AI 结果处理原则

所有 AI 输出都必须经过用户确认。

推荐动作：

- 插入当前位置。
- 追加到章节末尾。
- 替换选中文本。
- 保存为资料。
- 保存为记忆候选。
- 复制。
- 放弃。

禁止默认动作：

- 自动覆盖正文。
- 自动写入设定。
- 自动标记 OOC 通过。
- 自动同步云端。
- 自动发布。

## 长篇记忆 V0 推荐数据结构

MemoryItem：

| 字段 | 含义 |
| --- | --- |
| `id` | 记忆条目 ID |
| `workId` | 所属作品 |
| `chapterId` | 来源章节 |
| `type` | 人物 / 事件 / 设定 / 伏笔 / 世界规则 |
| `title` | 标题 |
| `content` | 记忆内容 |
| `sourceText` | 来源原文片段 |
| `sourceRange` | 来源位置 |
| `confidence` | 置信度 |
| `status` | draft / confirmed / rejected / stale |
| `createdBy` | user / ai |
| `createdAt` | 创建时间 |
| `updatedAt` | 更新时间 |

## 近期推荐任务

1. 先完成 Prompt 可编辑 / 导入。
2. 再完成章节统计联动。
3. 然后设计 Memory V0 本地结构。
4. 最后再接模型辅助抽取。

这样可以让长篇记忆成为产品主线，但不牺牲当前本地闭环的稳定性。
