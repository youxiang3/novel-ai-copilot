# 长篇记忆与 Prompt 策略

## 战略判断

用户选择长篇记忆作为官方工具的最高战略优先级。原因是长篇小说创作的核心痛点不是单次生成，而是长期上下文、人物一致性、设定延续和伏笔回收。

但当前阶段不直接实现完整 MemoryExtractionSkill。正确路径是先做长篇记忆 V0 前置能力，建立可确认、可追溯、可修正的记忆条目。

当前没有真实模型调用，没有向量库，没有后端记忆持久化。

## 长篇记忆分期

### V0：本地记忆前置能力

目标：让用户能看到、确认和维护“本书的重要记忆”。

能力范围：

- 章节摘要。
- 事件摘要。
- 人物状态摘要。
- 伏笔 / 设定摘录。
- 来源章节记录。
- 用户确认后写入。
- 标记记忆来源和更新时间。

V0 不承诺：

- AI 已记住全书。
- 自动长期上下文加载。
- 自动 OOC 检查。
- 自动纠错。
- 真实模型抽取。

### V1：模型辅助抽取

目标：接入真实模型，辅助从章节中提取候选记忆。

流程：

1. 用户选择章节或当前正文。
2. 系统构造 Prompt。
3. 模型返回候选记忆。
4. 用户确认、修改或拒绝。
5. 确认后的条目写入 MemoryItem。

原则：

- AI 只生成候选。
- 用户确认后才写入。
- 记忆条目必须有来源。
- 不自动覆盖已有设定。

### V2：MemoryExtractionSkill

目标：形成正式长篇记忆抽取能力。

能力：

- 人物、事件、设定、伏笔、世界规则分类抽取。
- 来源章节和原文片段引用。
- 置信度。
- 过期检测。
- 与故事线图、人物关系图、OOC 检查联动。

V2 依赖：

- 稳定章节数据。
- 后端持久化。
- Prompt 上下文结构稳定。
- 用户确认机制。

## OOC 与长篇记忆的关系

OOC 检查不应早于长篇记忆成熟。

OOC 至少依赖：

- 人物设定。
- 人物关系。
- 历史行为。
- 当前章节内容。
- 已发生事件。
- 世界规则。
- 长篇记忆。

如果没有这些数据，OOC 检查容易变成泛泛而谈。因此当前 OOC 应显示为原型展示或模型待接入。

## Prompt 策略

用户选择 Prompt 最终进入 Prompt Inspector。

分期如下：

### P0.5：本次 Prompt 可编辑

能力：

- 默认 Prompt 展示。
- 用户本次编辑 Prompt。
- 导入 `.txt` / `.md` Prompt。
- 恢复默认。
- 复制当前 Prompt。
- 不保存为模板。
- 不修改提示词库。
- 不接真实模型。

适用场景：

- 外部网页 AI。
- 临时实验。
- 快速调整生成要求。

### P1：Prompt 轻工作流

能力：

- 展示上下文来源。
- 展示 Prompt 摘要。
- 用户补充要求。
- 生成前确认。
- 结果回填后用户选择插入、复制或放弃。
- 记录最近若干次 Prompt。

仍不做：

- 复杂模板库。
- 模板变量编辑器。
- Prompt 市场。
- 自动覆盖正文。

### P2：Prompt Inspector

能力：

- 显示 Prompt 完整内容。
- 显示上下文来源。
- 显示引用的章节、人物、世界规则和记忆条目。
- 显示变量替换结果。
- 显示 token 估算。
- 显示风险提示，例如上下文不足、记忆过期、引用冲突。
- 支持调试 Prompt。

Prompt Inspector 是正式 AI 能力接入后的调试和透明层，不应在数据结构未稳定前强行实现。

## AI 结果处理原则

所有 AI 输出都必须经过用户确认。

推荐动作：

- 插入当前位置。
- 追加到章节末尾。
- 替换选中文本。
- 保存为资料。
- 保存为记忆候选。
- 复制。
- 放弃。

禁止默认动作：

- 自动覆盖正文。
- 自动写入设定。
- 自动标记 OOC 通过。
- 自动同步云端。
- 自动发布。

## 长篇记忆 V0 推荐数据结构

MemoryItem：

| 字段 | 含义 |
| --- | --- |
| `id` | 记忆条目 ID |
| `workId` | 所属作品 |
| `chapterId` | 来源章节 |
| `type` | 人物 / 事件 / 设定 / 伏笔 / 世界规则 |
| `title` | 标题 |
| `content` | 记忆内容 |
| `sourceText` | 来源原文片段 |
| `sourceRange` | 来源位置 |
| `confidence` | 置信度 |
| `status` | draft / confirmed / rejected / stale |
| `createdBy` | user / ai |
| `createdAt` | 创建时间 |
| `updatedAt` | 更新时间 |

## 近期推荐任务

1. 先完成 Prompt 可编辑 / 导入。
2. 再完成章节统计联动。
3. 然后设计 Memory V0 本地结构。
4. 最后再接模型辅助抽取。

这样可以让长篇记忆成为产品主线，但不牺牲当前本地闭环的稳定性。
