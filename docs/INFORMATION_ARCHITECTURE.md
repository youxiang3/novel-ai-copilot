# 信息架构

## App Shell 结构

NovelAI Copilot 当前采用统一 App Shell：

- 左侧全局导航：承载主要模块入口。
- 顶部栏：承载当前作品 / 状态 / 搜索 / 用户区等全局操作。
- 中间主工作区：根据当前模块展示作品库、作品总览、章节编辑、资料管理等内容。
- 右侧上下文区：展示当前作品详情、官方工具状态、故事线辅助面板或当前页面辅助信息。

页面不应创建割裂的新壳层。新增功能优先放入现有 App Shell 的主工作区或右侧上下文区。

## 左侧导航

当前主要导航包括：首页、创作台、作品库、资料库、写作工具、灵感记录、模型设置、外观主题、导出发布、工具箱、提示词库和故事线图。

部分导航当前是入口或原型页面，不代表所有能力已完整接入。

## 顶部栏

顶部栏用于展示当前作品选择、当前作品状态、章节范围或工作区上下文、搜索入口、同步 / 云端状态入口、通知、帮助和用户区。

当前同步相关展示以本地状态和 UI 占位为主，不代表真实云同步已完成。

## 作品库

作品库是多作品项目管理入口，核心区域包括：

- 顶部统计卡片：正式作品、临时草稿、总字数、本月更新、云端同步状态。
- 搜索 / 筛选 / 排序区：标题、笔名、标签、题材、平台、进度、字数范围和排序。
- 卡片视图：适合浏览作品封面、简介、进度和状态。
- 列表视图：适合批量对比作品数据。
- 右侧官方工具面板：根据当前选中作品展示工具启用状态。
- 右侧当前作品详情：展示选中作品的身份、进度、平台、同步状态和已启用工具。

作品库数据来自 `localStorage:yixie-works-library-v1`。

## 作品总览

作品总览是进入单本作品后的默认驾驶舱，展示身份区、核心信息区、全局大纲区、创作进度区、作品资料摘要区、章节入口区和官方工具状态区。

作品总览读取作品库同一份本地作品数据。

## 章节管理

章节管理当前属于正式写作工作台的一部分，支持新建章节、编辑章节标题和正文、保存章节、字数统计、删除章节和发布标记。

当前章节统计与作品库统计仍有前端字段联动风险，后续需要独立章节模型和后端统计服务。

## 设定库

设定库用于承载作品资料、人物、世界观和其他创作素材。当前部分能力以本地展示和入口形式存在，后续应与作品数据模型、章节上下文和故事线图打通。

## 灵感

灵感记录用于保存脑洞、短画面、卡点想法和后续可转化为作品或章节的素材。当前不作为后端持久化能力承诺。

## 世界观图谱

世界观图谱是作品设定的可视化入口。当前作为创作工作台中的功能入口或本地原型展示，后续可接入设定实体、人物关系和章节引用。

## 故事线图

故事线图用于管理章节事件、人物、地点、钩子和伏笔。当前本地数据 key 为：

`localStorage:yixie-storyline-data-v1-${workId}`

本轮文档不扩展 StoryGraphCenter 逻辑。

## 官方工具状态区

官方工具包括黄金第一章、人物关系图、钩子 / 伏笔线性图、长篇记忆、OOC 检查、卡文急救和发布准备。

每个工具应区分功能状态和接入状态。功能状态说明用户是否启用或检查是否通过；接入状态说明当前是原型展示、模型待接入、本地可用、已接入模型或云端待同步。

## 右侧当前作品详情区

右侧详情区跟随当前选中作品变化，展示封面、作品名、题材、状态、目标字数、当前字数、完成进度、章节进度、发布平台、最近任务、云端同步状态和已启用工具。

## 跳转关系

- 首页 / 脑洞入口 -> 新建作品确认流程 -> 作品库 / 作品总览。
- 作品库卡片 / 列表 -> 作品管理弹窗。
- 作品库卡片主操作 -> 进入作品写作或查看作品。
- 作品库选中作品 -> 右侧当前作品详情更新。
- 作品库进入作品 -> 默认打开作品总览。
- 作品总览 -> 章节管理 / 正式写作工作台。
- 作品总览 -> 故事线图入口。
- 作品总览 -> 官方工具入口。
- 故事线图 -> 读取当前作品对应的本地故事线数据。
